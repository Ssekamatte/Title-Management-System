Manage Static Information

 

The process of updating status of the decisions of Summit, Extraordinary Summit, Council of Ministers, and Extraordinary Council of Ministers is the same. The manage static information option allows the user to view and update the status of a given record in the database. Selecting this option pops-up the following screen. The step-by-step instruction is given below.

From the main menu select the type of decision/directive (Summit, Extraordinary Summit, Council of Ministers, and Extraordinary Council of Ministers) that you would like to update by clicking on the manage static information option. A screen with the search panel will be presented. The Data Grid will appear with existing information.

Select the data you would like to update and press the Search Button. The data shall populate the grid as shown below. The data shall display in the grid with color codes (Traffic Lights) in the last column to symbolize the status of the decision. Green shows that the Decision fully implemented or Directive fully adhered to; Yellow shows that the decision has been partially implemented or implementation still on-going; Red shows that the Decision has not been implemented /No action taken to date and Blue shows Conscious Decisions/Directives.

Search Panel

The Search Panel is used to quickly search for data from the database using different parameters. It is important to note that the search panel parameters are conjunctive in that the items entered into the different search parameters are searched for together as a logical And. As an example, if the user types 'Nile Perch' in the free text search text box and enters 37 in the Meeting No. search field, then the search will return only those records that have both conditions as true.

When a user is interested in searching for an item captured on in a given date range then “Start Date” represents the starting date within the range and “End Date” represents the ending date within the range.

Manage Static Information

In this system the components that the ordinary users cannot modify are referred to as Static Data. These are predefined details decided upon by Summit, Extraordinary Summit, Council of Ministers, Extraordinary Council of Ministers, and the Sectoral Council or predetermined by the Common Market Protocol, the Development Strategy or any other instrument that guides the EAC Secretariat.

As an example, for Decisions, the Decision Reference, Description, Responsibility Centre, Time frame, Budgetary Implications, and the Performance Indicators are static data. These are captured by the systems Administrators. The users see them but can't change them.

Update Progressive Information

Progressive data are the attributes of the system that the user updates. For instance, for Decisions of Council, an ordinary user can only change the Implementation Status of the decision (Traffic light), Implementation Status Narrative and the Status / Comments. These are what are termed progressive data.

The data populates the grid as shown below. The data shall display in the grid with color codes (Traffic Lights) in the second column to symbolize the status of the decision. Green shows that the Decision fully implemented or Directive fully adhered to; Yellow shows that the decision has been Partially implemented or implementation still on-going; Red shows that the Decision has not been implemented /No action taken to date and Blue showing Continuous decisions/directives.

The process of adding a new decision/directive or updating status of the decisions of Summit, Extraordinary Summit, Council of Ministers, Extraordinary Council of Ministers, and the Sectoral Council is the same.
